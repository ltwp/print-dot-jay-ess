History.

Printing from the browser is a messy affair. Each browser has its own user agent stylesheet, which competes with the html of a page itself and the user's printer defaults to spit something out to a page of paper.

That's why there are now media queries like @media print and @page, in addition to lessons (one, two) on how to use them and best practices for going from web to printed matter.

print.js is my attempt to tidy up a small corner of this landscape. It is approaching version 1.0.

print.js

print.js came out of HTML Output at RISD. In the class, I noticed a number of people (myself and John Caserta especially) producing what I call "web tapestries". These are patterns, some interactive, using simple web elements to create textures and repeats in the browser, inspired by standards in textiles and print industries.

Some examples:





I wanted to create a way to print these pages out that reflected how they are in the browser: full-bleed, correctly aligned (as they were created in the browser), and beautiful. Just hitting ⌘P didn't do it, though. I had to create a @media print stylesheet to do everything up right. But I figured I could do this in a standardized way; most of the web tapestries followed a structure of simple nested divs.

With a few people making this, a standard emerged. And with a standard comes automated editability. I went about creating a javascript file that, through jQuery, would generate the stylesheet you need for print.

What it does.

The gist of print.js is that you stick it in your head (alongside jQuery), specify the size of paper you want to output to, and then reload the page. Then hit print. It should be beautiful. But only if you follow some ground rules:

  • All other javascript in the page must be in the head or an external file. Javascript used to populate the page cannot be in the body.
  • If you have a wrapper that is second only to the body in terms of page hierarchy, make it named 'wrapper' or 'container'.
  • The assumed structure is body > "wrapper" > pattern containers > pattern elements. They are assumed to be divs as of now.
  • Your CSS should be in the document rather than as an external stylesheet.
  • Start your document with a <!DOCTYPE HTML> declaration. jQuery needs this for document height detection.
  • Be aware that the edges of your pattern will be cut off. You can leave those beautiful selvedges for other places.
  • Print from Google Chrome, and I suggest doing full bleed (i.e. no margins).

Installing.

Some issues remain, no doubt, but this is a work in progress...

To install print.js on a page, put the print.js file in the same directory and this at the end of your document's <head>:

<script src="http://code.jquery.com/jquery-latest.js"></script>
<script src="print.js"></script>

That is it! Let me know if you have any issues. lukas@brown.edu.
